## 3. WHAT WE'RE PITCHING

### 3.1 Phase 1 - Core POS (Immediate Build)

| Feature | Description |
|---------|-------------|
| **Order Management** | Dine-in (table map), takeaway, delivery - unified workflow |
| **Menu Engine** | Categories, items, modifiers, combos, photos, multi-language |
| **Kitchen Integration** | KDS screens or printer routing by station |
| **Payments** | Cash, card, mobile wallet, split checks, tips |
| **Staff Management** | Login, roles, permissions, manager overrides |
| **End-of-Day** | Z-report, cash reconciliation, sales summary |
| **QuickBooks Sync** | Auto-push daily sales, taxes, tips to QB Online |
| **Reporting** | Sales by item/category/time/server, exportable reports |
| **Receipt Printing** | Customizable receipts with logo, tax breakdown |
| **Offline Mode** | Core operations work without internet, sync when back online |

**Differentiators vs Toast / Square / Clover:**
- No per-terminal monthly licensing fees
- No forced payment processor lock-in
- Client owns the system and data fully
- Custom-built around THEIR workflow, not a generic template
- QuickBooks integration included - not a paid add-on
- No revenue-sharing on online orders

### 3.3 Technical Architecture (If Client Asks)

```
CLIENT DEVICES                    CLOUD INFRASTRUCTURE
=============                    ====================

[iPad/Tablet POS] ----\
[Kitchen KDS]     -----+----->  [FastAPI Backend]  --->  [PostgreSQL DB]
[Manager Tablet]  ----/              |
                                     |---> [QuickBooks API]
[Owner Mobile App] --------------->  |---> [Payment Gateway]
                                     |---> [Receipt Printer Service]
[Online Ordering] ---------------->  |---> [Cloud Storage (Receipts/Reports)]
```

- **Frontend:** React - runs on any device with a browser (iPad, Android tablet, PC)
- **Backend:** FastAPI + PostgreSQL - fast, scalable, battle-tested
- **Hosting:** Cloud VPS (or on-premise server if client prefers)
- **Payments:** Integrates with their existing processor or Stripe Terminal
- **QuickBooks:** Official API with OAuth2 - secure, real-time capable

## 6. RISKS & OBJECTIONS - BE READY

| Objection | Response |
|-----------|----------|
| "Why not just buy Toast/Square?" | You'll pay $70-150/terminal/month forever, locked into their processor at higher rates, no customization, and QB integration is a paid add-on. With us, you own it. |
| "What if you disappear?" | The code is yours. It runs on standard tech. Any developer can maintain it. We'll provide full documentation and source code. |
| "Can it handle our volume?" | FastAPI handles thousands of requests/second. PostgreSQL powers companies 100x your size. This stack doesn't break. |
| "What about PCI compliance?" | We never store card data. Payments go directly through the processor's certified terminal. We only receive confirmation tokens. |
| "We need it yesterday." | We can do an MVP in 3-4 weeks: orders, payments, kitchen printing, basic reporting. Then iterate. |
| "What about updates and bugs?" | Included in the support plan. Plus, since it's web-based, updates deploy instantly - no app store waits. |
